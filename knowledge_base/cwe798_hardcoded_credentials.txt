Vulnerability Name:
Hardcoded Credentials

CWE:
CWE-798

Description:
Credentials, passwords, API keys, and tokens are embedded directly in source code. If source code is exposed, attackers gain unauthorized access.

Risk Level:
High

Detection Indicators:
- password = "..."
- API_KEY = "..."
- token = "..."
- secret = "..."

Secure Coding Guidance:
- Use environment variables
- Use secret managers
- Store credentials outside source code

Repair Strategy:
Move secrets into environment variables or external secret stores.

Vulnerable Example:

API_KEY = "abc123"

Secure Example:

API_KEY = os.getenv("API_KEY")
